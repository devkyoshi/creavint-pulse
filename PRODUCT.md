# Product

## Register

product

## Users

Internal staff at Creavint: admins, site managers, content reviewers, and analysts. They use this tool daily to operate an automated content monetization network — provisioning Hugo sites, monitoring content pipelines, reviewing AI-drafted articles, and tracking AdSense / GA4 / GSC performance. Context is high-focus operations: checking what's broken, approving or rejecting work, reading numbers. Speed and clarity matter more than visual pleasure.

## Product Purpose

Creavint Pulse is an internal operations dashboard for running automated content monetization at scale. It provisions static blog sites, drives an LLM content pipeline, surfaces SEO and analytics data, and feeds revenue signals back into content decisions. Phase 1a goal: 3–5 pilot sites reaching `monetized` state.

## Brand Personality

Focused, precise, reliable. A trusted instrument — no surprises, no noise. The tool disappears into the task; information is the foreground.

## Anti-references

- Generic SaaS admin: cream/warm-neutral backgrounds, identical metric card grids, over-templated layouts
- Glassmorphism, gradient text, decorative motion
- Marketing/editorial style: big display typography, scroll-driven sections, brand-forward layout
- Bland enterprise gray: featureless, low-contrast, no intentionality

## Design Principles

1. **Information is the product.** Every visual choice either helps the user read data faster or gets out of the way.
2. **Earned familiarity.** Use standard affordances. Operators in flow should never pause at a UI element.
3. **State speaks first.** Color, weight, and position signal system state — not decoration.
4. **Density when needed, space when earned.** Dense tables where scanning is the task; generous spacing where attention needs to land.
5. **No cream.** The palette should feel cool, calibrated, and intentional — not warm-default or AI-generic.

## Accessibility & Inclusion

WCAG AA minimum. Body text ≥4.5:1 contrast. Interactive states (hover, focus, active) must be distinct and keyboard-navigable. Reduced-motion fallbacks for any animation added.
